# 招聘信息爬虫与 AI 评分系统

开发者规范文档 · v1.0 · 面向 Codex / AI 编码代理
目标平台：liepin.com（猎聘网） · AI 引擎：DeepSeek Chat API · 输出：PDF 报告

---

# 1. 项目概述

该系统会自动从 liepin.com（中国大型招聘平台“猎聘网”）抓取职位信息，并通过 DeepSeek Chat API 从多个维度评估职位真实性，最终生成一份带排序的 PDF 报告。

系统目标是过滤掉“幽灵岗位（ghost listings）”—— 即已经发布但长期无人维护的招聘信息，并优先筛选真正有招聘需求的岗位。

核心问题：

许多招聘网站上的职位已经数月无人更新，HR 也从不回复。该系统通过量化多个能够反映职位“新鲜度”和“企业招聘诚意”的代理信号，再交由 AI 做整体判断。

## 1.2 liepin.com 搜索 URL 结构

搜索 URL 结构如下（通过分析网站验证）：

```text
https://www.liepin.com/zhaopin/?key={keyword}&city={city_code}&pageNo={page}
```

示例：

```python
# 上海 Python，第 1 页
https://www.liepin.com/zhaopin/?key=python&city=010200&pageNo=0

# 上海 Java，第 2 页

# 上海 C#（# 需 URL 编码）

```

说明：

* `pageNo` 从 0 开始计数
* 城市代码：

  * 上海 = 010200
  * 北京 = 010100
  * 杭州 = 080200
  * 深圳 = 070200
* 所有配置集中放在 `config.py` 中，方便扩展

# 2. 系统架构

## 2.1 四层流水线架构

### ■ 第 1 层 —— 爬虫（Crawler）

使用 Playwright（无头 Chromium）渲染 React 单页应用（SPA），从列表页提取职位 ID，再抓取详情页。

整个过程中使用反检测策略。

---

### ■ 第 2 层 —— 存储（Storage）

使用 SQLite 保存：

* 原始职位数据
* AI 评分结果

通过 `job_id` 主键去重。

使用 `crawl_log` 表支持增量爬取。

---

### ■ 第 3 层 —— AI 评分（AI Scoring）

将职位 JSON 数据发送给 DeepSeek Chat API。

AI 返回：

* 各维度评分
* 最终结论
* 风险标记
* 简短分析说明

---

### ■ 第 4 层 —— 输出（Output）

按总分排序职位。

高于阈值的职位通过 ReportLab 生成 PDF 报告，包括：

* 可视化分数条
* 风险警告提示

# 3. 爬虫模块

## 3.1 为什么使用 Playwright（而不是 requests）

猎聘网列表页属于 React SPA。

普通 HTTP GET 返回的只是空壳 HTML，不包含职位数据 —— 必须执行 JavaScript。

Playwright 会驱动真实 Chromium 浏览器，并等待：

```python
networkidle
```

后再提取 DOM。 

---

职位详情页中包含：

```html
<script type="application/ld+json">
```

其中嵌入了 Schema.org 的 JobPosting 数据。

这里包含：

* 完整 JD
* 发布时间
* 更新时间
* 公司信息
* 薪资

因此直接：

```python
json.loads()
```

解析即可，无需脆弱 CSS 选择器。

## 3.3 list_page.py —— 核心逻辑

```python
async def scrape_list_page(page, keyword: str, page_no: int) -> list[str]:
    """返回某搜索结果页中的职位详情 URL 列表"""

    url = (
        f"https://www.liepin.com/zhaopin/"
        f"?key={keyword}&city={SEARCH_CONFIG['city_code']}&pageNo={page_no}"
    )

    await page.goto(url, wait_until="networkidle", timeout=30000)

    await human_like_scroll(page)  # 反检测滚动

    links = await page.query_selector_all("a[href*='/a/']")

    job_ids = []

    for link in links:
        href = await link.get_attribute("href")

        if match := re.search(r'/a/(\d+)', href):
            job_ids.append(match.group(1))

    return list(set(job_ids))  # 去重
```

## 3.4 detail_page.py —— 核心逻辑

```python
def parse_detail_page(html: str, job_id: str) -> dict:
    """从职位详情 HTML 中提取结构化数据"""

    soup = BeautifulSoup(html, "lxml")

    job = {"job_id": job_id}

    # 1. 优先解析 ld+json
    for tag in soup.find_all("script", type="application/ld+json"):
        try:
            data = json.loads(tag.string)

            if data.get("@type") == "JobPosting":
                job.update(extract_from_ld_json(data))
                break

        except (json.JSONDecodeError, AttributeError):
            continue

    # 2. 补充更新时间字段
    el = soup.select_one(".time-factor-wrap")

    if el and (m := re.search(r"\d{4}-\d{2}-\d{2}", el.text)):
        job["last_updated"] = m.group()

    # 3. 计算衍生字段
    if job.get("last_updated"):
        delta = date.today() - date.fromisoformat(job["last_updated"])
        job["days_since_update"] = delta.days

    return job
```

## 3.5 anti_detect.py —— 反检测机制

| 模块             | 说明                                                 |
| -------------- | -------------------------------------------------- |
| 动态渲染           | `wait_for_load_state("networkidle")` 等待 JS 完成      |
| WebDriver 指纹隐藏 | playwright-stealth 修补 navigator.webdriver 等 20+ 信号 |
| 行为模拟           | 随机鼠标移动 + 页面滚动 + 4~9 秒随机等待                          |
| User-Agent     | 使用 fake-useragent 随机生成 Windows/Mac Chrome UA       |
| Session / 登录   | 手动导出 cookies.json 后通过 `context.add_cookies()` 注入   |
| CAPTCHA（极少）    | 截图 + 暂停提醒；个人使用频率下几乎不会触发                            |

# 5. AI 评分模块

## 5.1 评分维度

| 维度                      | 满分   | 评分逻辑                     |
| ----------------------- | ---- | ------------------------ |
| 活跃度（Activity Freshness） | 30 分 | ≤7 天满分；≥60 天为 0          |
| JD 真实性                  | 25 分 | 技术栈明确、职责 ≥4 条、≥300 字得高分  |
| 公司可信度                   | 20 分 | 已认证 + logo + ≥500 人公司得高分 |
| 薪资合理性                   | 15 分 | 范围合理 + 标明年终奖月份           |
| 其他信号                    | 10 分 | HR 直招、明确办公地点、具体福利        |

## 5.4 API 调用（scorer.py）

```python
from openai import OpenAI

client = OpenAI(
    api_key=DEEPSEEK_API_KEY,
    base_url="https://api.deepseek.com/v1"
)

def score_job(job: dict) -> dict:
    """调用 DeepSeek 并返回评分 JSON"""

    payload = json.dumps(job, ensure_ascii=False, indent=2)

    resp = client.chat.completions.create(
        model="deepseek-chat",
        messages=[
            {"role": "system", "content": SYSTEM_PROMPT},
            {
                "role": "user",
                "content": USER_PROMPT_TEMPLATE.format(job_json=payload)
            }
        ],
        temperature=0.1,
        max_tokens=400,
        response_format={"type": "json_object"}
    )

    return json.loads(resp.choices[0].message.content)
```

说明：

* DeepSeek API 与 OpenAI SDK 兼容
* `temperature=0.1` 保证 JSON 稳定输出
* `response_format={"type":"json_object"}` 强制 JSON 模式

# 9. Codex 实现注意事项

以下设计为有意决定，请勿随意修改：

* 爬虫必须全程使用 asyncio
* Playwright Python API 原生支持 async
* ld+json 解析器必须兼容：

  * 单 dict
  * dict 数组
* `days_since_update` 必须在抓取时计算，而非评分时
* `response_format={"type":"json_object"}` 可避免正则清洗 JSON
* 推荐 SQLite WAL 模式：

  ```sql
  PRAGMA journal_mode=WAL
  ```
* 所有延迟必须：

  ```python
  random.uniform(delay_min, delay_max)
  ```

  禁止固定 sleep
* `cookies.json` 格式：

  ```json
  [
    {
      "name": "...",
      "value": "...",
      "domain": "...",
      "path": "..."
    }
  ]
  ```
