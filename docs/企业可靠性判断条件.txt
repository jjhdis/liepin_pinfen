目前代码里已经先落地了第一版筛选逻辑，但不是把本文所有条件一次性全部实现，而是先挑了当前最容易稳定量化的几条：

- 公司规模小或未知

- 命中不足 2 条 -> `need_company_check = 0`

后续在知乎抓取接入后，再继续补更细的规则，例如：

- 公司成立时间短
- 更可靠的平台异常信号
- 更细的薪资异常判断
- 更细的公司规模标准化

你的思路非常清晰：**大公司部门多、情况复杂，一个负面评价不代表全貌，而且大公司通常有正规HR体系，不太会明目张胆地骗人。** 所以真正需要深挖的，是那些“听起来不错、但总觉得哪里不对”的中小公司。

下面我给出**5个触发“深度调查”（爬知乎/看评价）的特征**，你可以把它们加到评分系统里——当一条招聘信息命中其中任意2条或以上，就自动标记为“需深度调查”，再去爬知乎。

---

## 触发深度调查的5个特征

### 特征1：公司规模 ≤ 50人（或查不到）
- **为什么可疑**：小公司缺乏制度约束，老板一言堂，更容易出现拖欠工资、不交社保、PUA等乱象。而且小公司在知乎上被曝光的概率低，一旦有人专门写回答，往往就是大坑。
- **可量化规则**：天眼查/企查查参保人数 < 20，或者BOSS上公司规模显示“0-20人”、“20-99人”且无官网。

### 特征2：公司成立时间 ≤ 2年
- **为什么可疑**：初创公司存活风险高，很多撑不过一年。而且新公司为了快速盈利，可能采用压榨、骗补、甚至违法手段。成立2年以上的公司至少证明它能活下来。
- **可量化规则**：公司成立日期距今 < 730天。

### 特征3：薪资明显高于市场水平20%以上（且公司规模小）
- **为什么可疑**：小公司如果开出远超行业标准的薪资，要么是画大饼（绩效根本拿不到），要么是骗局（培训贷、刷单）。正规小公司往往薪资偏低或持平市场，靠股权/画饼吸引人，不会突然高薪。
- **可量化规则**：你的评分系统里，薪资合理性得分偏低（比如仅得2-5分/10分），但同时薪资数字又很高（如“20k-30k”而行业平均15k）。

### 特征4：JD中出现2个以上“负面词库”中的**严重类词汇**
- **为什么可疑**：比如“押金、培训贷、刷单、传销、纯提成、无底薪、扣押身份证”等。这些词一旦出现，几乎100%是坑，不需要去知乎查，可以直接放弃。但如果你想看看到底多离谱，可以查知乎“公司名+骗局”。
- **可量化规则**：用你已有的负面词库匹配，命中≥2个严重类词（可单独建一个“严重词子集”）。

### 特征5：招聘信息存在“平台异常信号”
- 例如：BOSS直聘上该岗位**发布超过30天**但HR一直活跃（僵尸岗），或者**同一公司一周内发布多个相似岗位**（HR刷KPI），或者**公司简介含糊**（“XX集团、XX控股”但查不到实际业务）。
- **为什么可疑**：这些信号说明公司要么根本不急着招人（拿求职者凑数），要么就是长期招不到人（说明口碑差、留不住人）。这时候去知乎搜一下，往往能找到真实评价。

## 组合使用建议

你可以写一段逻辑：

```python
def need_deep_check(job_info):
    score = 0
    if job_info.employee_count <= 50: score += 1
    if job_info.company_age_days < 730: score += 1
    if job_info.salary_abnormal and job_info.is_small_company: score += 1
    if job_info.jd_contains_severe_keywords >= 2: score += 1
    if job_info.platform_anomaly: score += 1
    return score >= 2   # 命中2条及以上，就去爬知乎
```

这样你既能避免对每一家公司都做深度调查，又能精准定位最需要警惕的目标。

## 补充说明：为什么大公司不查？

你说得对，大公司（比如1000人以上）即使有负面评价，也常常是某个部门的问题，整体还是规范的。比如“阿里加班严重”是普遍现象，但阿里不会拖欠工资或搞培训贷。所以：

- **大公司**：用评分系统的其他维度（时间、JD质量）就够了，不需要专门爬知乎。
- **中小公司**：必须深度调查，尤其要查知乎、看准网、脉脉等员工匿名吐槽。

## 一个实操技巧

爬知乎比较麻烦，你可以先手动快速验证：  
在浏览器地址栏输入 `site:zhihu.com "公司名" 坑` 或 `site:zhihu.com "公司名" 怎么样`，看有没有高赞回答。如果前两页没有相关讨论，说明这家公司可能太小众，或者还没人曝光（不代表安全，但至少没有公开大雷）。

如果你愿意，我可以帮你写一个**半自动脚本**：用DuckDuckGo搜索 `site:zhihu.com 公司名 负面`，抓取前几条结果摘要，再用AI判断是否有价值。这样比爬知乎全站简单得多，而且不违反知乎反爬。需要的话告诉我。
