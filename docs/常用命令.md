# 常用命令

这份文档整理当前项目最常用的命令，包括：

- 获取猎聘 Cookie
- 运行 `list` / `detail` 爬虫
- 运行清洗、企业增强、评分
- 启动本地 dashboard

---

## 1. 获取 Cookie

### 1.1 半自动登录并导出 Cookie

默认会导出到 `cookies/` 目录，文件名格式类似：

```text
cookie_时间_手机号_liepin.json
```

命令：

```powershell
.\.venv\Scripts\python.exe .\tools\refresh_liepin_cookies.py --phone 13800138000
```

### 1.2 手动指定输出路径

```powershell
.\.venv\Scripts\python.exe .\tools\refresh_liepin_cookies.py --phone 13800138000 --output .\cookies\cookie_custom_13800138000_liepin.json
```

### 1.3 从浏览器复制的原始 Cookie 字符串转换

```powershell
.\.venv\Scripts\python.exe .\tools\convert_cookie_string.py "name1=value1; name2=value2" --phone 13800138000
```

## 3. 爬虫主命令

### 3.1 抓列表页

当前默认每页只保留前 `20` 条职位进入待抓详情队列。

```powershell
.\.venv\Scripts\python.exe .\main.py list --keyword python --cookie-file cookie_20260603_090000_13800138000_liepin.json
```

### 3.2 抓详情页

当前最保守基线建议：

- 单个 Cookie
- 单个 keyword
- 单轮最多 `20` 条 detail

```powershell
.\.venv\Scripts\python.exe .\main.py detail --keyword python --max-detail 20 --cookie-file cookie_20260603_090000_13800138000_liepin.json
```

### 3.3 显式覆盖列表页保留条数

```powershell
.\.venv\Scripts\python.exe .\main.py list --keyword java --store-top-n 20 --cookie-file cookie_20260603_091500_13900139000_liepin.json
```

## 6. 猎聘消息检查 DLC

当前已新增独立消息检查入口：

- `check_liepin_messages.py`

用途：

- 直接请求猎聘联系人接口
- 检查当前账号是否有未读消息
- 记录最近联系人和最近消息时间
- 支持显式指定 Cookie 文件

### 6.1 检查单个 Cookie 的消息状态

```powershell
.\.venv\Scripts\python.exe .\check_liepin_messages.py --cookie-file cookie_20260603_094006_15317685379_liepin.json
```

### 6.2 手动指定 Cookie profile id 和账号标签

```powershell
.\.venv\Scripts\python.exe .\check_liepin_messages.py --cookie-file cookie_20260603_094006_15317685379_liepin.json --cookie-profile-id liepin_a --account-label liepin_15317685379
```

### 6.3 只做请求和输出，不写数据库

```powershell
.\.venv\Scripts\python.exe .\check_liepin_messages.py --cookie-file cookie_20260603_094006_15317685379_liepin.json --dry-run
```

## 8. 自动轮换当天 Cookie

当前已新增一个独立轮换入口：

- `rotate_liepin_cookies.py`

用途：

- 只扫描 `cookies/` 目录里当天生成的猎聘 Cookie
- 每个 Cookie 跑一轮 detail
- 每轮默认 `25` 条
- 跑完自动切到下一份 Cookie
- 默认每份 Cookie 只用一次

### 7.1 按默认策略轮换

```powershell
.\.venv\Scripts\python.exe .\rotate_liepin_cookies.py --keyword python
```

说明：

- `--keyword` 不是必填
- 不指定 `--keyword` 时，会从 `jobs` 表中按创建顺序处理所有 `detail_status = 'pending'` 的职位

### 7.2 不指定 keyword，直接轮换全部 pending 职位

```powershell
.\.venv\Scripts\python.exe .\rotate_liepin_cookies.py --max-cookies 3
```

适用场景：

- 已经分别执行过 `python` / `java` / `全栈` 的 `list`
- 现在想让当天的 3 份 Cookie 直接轮换跑 detail
- 不需要再按关键词拆开跑

### 7.3 指定每个 Cookie 跑多少条

```powershell
.\.venv\Scripts\python.exe .\rotate_liepin_cookies.py --keyword java --per-cookie-detail 25
```

如果不指定 `keyword`，也可以这样写：

```powershell
.\.venv\Scripts\python.exe .\rotate_liepin_cookies.py --max-cookies 3 --per-cookie-detail 25
```

### 7.4 只使用当天前 3 份 Cookie

```powershell
.\.venv\Scripts\python.exe .\rotate_liepin_cookies.py --keyword 全栈 --max-cookies 3
```

### 7.5 只打印计划，不实际执行

```powershell
.\.venv\Scripts\python.exe .\rotate_liepin_cookies.py --keyword c# --dry-run
```

如果想预览“全部 pending”的轮换计划：

```powershell
.\.venv\Scripts\python.exe .\rotate_liepin_cookies.py --max-cookies 3 --dry-run
```

### 7.6 指定 Cookie 日期

```powershell
.\.venv\Scripts\python.exe .\rotate_liepin_cookies.py --keyword python --cookie-date 20260603
```

## 9. 当前建议

当前阶段建议把下面这组作为默认运行单元：

1. `1 个 Cookie`
2. `1 个 keyword`
3. `1 轮 20-25 条 detail`

当前不要把下面这些当默认策略：

- 单 Cookie 常规跑 `25-30` 条
- 单页保留 `30-40` 条列表尾部职位
- 命中 `expired / wrong_page` 后继续跑当前批次
