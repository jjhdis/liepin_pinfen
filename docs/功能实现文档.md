## 1. 项目概览

这是一个**猎聘（liepin）招聘数据抓取、清洗、企业风险调查、AI 评分**的管道式系统。核心能力：

- 通过 Playwright 模拟浏览器抓取猎聘职位列表和详情页
- 离线清洗原始 HTML/JSON，补齐薪资/城市/学历等字段
- 对可疑公司自动搜索知乎，收集负面证据
- 调用 DeepSeek API 对职位进行五维度评分
- 提供 Web Dashboard 监控全链路状态
- 多 Cookie 账号轮换，冷却复用机制

**技术栈**：Python 3.9 + Playwright + SQLite + curl_cffi + DeepSeek API + 原生 http.server

     │

     ▼                          ▼

     │                          │
     ▼                          ▼
  company_enriched 表         scores 表

### 执行顺序

| 顺序 | 阶段 | 入口 | 输入 | 输出 |
|------|------|------|------|------|
| 1 | 列表抓取 | `main.py list` | 关键词 | `jobs` (job_id + detail_url) |
| 2 | 详情抓取 | `main.py detail` | `jobs` pending | `jobs` (title, salary, jd_text, ...) |
| 3 | 清洗 | `clean_jobs.py` | `jobs` success | `jobs_cleaned` |
| 4 | 企业增强 | `company_enrich.py` | `jobs_cleaned` need_company_check | `company_enriched` |
| 5 | AI 评分 | `score_jobs.py` | `jobs_cleaned` ready | `scores` |

**串联入口**：`postprocess_pipeline.py` 按 3→4→5 顺序执行。`rotate_liepin_cookies.py` 每批详情跑完后可自动触发后处理。

## 4. 模块详解

### 4.1 抓取层

```
crawler/
├── browser.py        # 浏览器启动、Cookie 注入、反检测脚本
├── list_page.py      # 搜索页打开、职位卡片提取
├── detail_page.py    # 详情页访问、ld+json 解析、异常分类
├── anti_detect.py    # 随机延时、人类行为模拟
└── zhihu_client.py   # 知乎搜索摘要抓取
```

#### `main.py` — 统一 CLI 入口

```
python main.py list --keyword python     → 列表抓取
python main.py detail --max-detail 25    → 详情抓取
```

**列表抓取流程**：
1. 打开浏览器 → 注入 Cookie → 访问搜索页
2. 提取 `<div class="job-card-pc-container">` 中非 `display:none` 的卡片
3. 取 `store_top_n` 条写入 `jobs` 表（status=pending）
4. 支持多页翻页（含随机 60-120s 间隔）

**详情抓取流程**：
1. 从 `jobs` 表取 `detail_status='pending'` 的记录（最多 `max_detail` 条）
2. 先访问搜索页做 warmup，再直接 goto 详情 URL
3. 解析页面中的 `<script type="application/ld+json">` JobPosting
4. 容错处理非法反斜杠、多行文本等非标准 JSON
5. 根据结果设置 `detail_status`：
   - 成功 → `success`，填充所有字段
   - blocked/login_required → 立即终止批次
   - expired/wrong_page → 删记录 + 终止批次
   - parse_failed/timeout → 标记 + 继续下一条

**风控节奏**：
- 启动冷却：120-180s
- 详情间隔：45-90s
- 每 10 条额外冷却 180-240s
- 单批默认 25 条

### 4.2 清洗层

```
cleaning/job_cleaner.py    # 核心清洗逻辑
clean_jobs.py              # CLI 入口
```

**清洗功能**：
- HTML 回填薪资（`ld+json` 的 baseSalary 覆盖不足）
- 从公司信息区补提公司名、公司规模
- 缺失 `jd_text` 时从正文区域提取
- 计算 `jd_length`
- 生成 `ai_input_json`（结构化输入）
- 生成 `quality_flags_json`（质量标记）
- 企业调查触发判定（5 条规则，见 4.4）

### 4.3 AI 评分层

```
ai/
├── prompts.py    # Prompt 模板和版本管理
├── parser.py     # AI 响应解析和校验
└── scorer.py     # API 调用封装

score_jobs.py     # CLI 入口
```

**五维度评分**：

| 维度 | 满分 | 评估内容 |
|------|:--:|------|
| score_activity | 25 | 职位发布时间、更新频率 |
| score_jd | 25 | JD 详细度、专业性 |
| score_company | 20 | 公司规模、企业风险证据 |
| score_salary | 20 | 薪资竞争力、结构透明度 |
| score_other | 10 | 加分项 |

**校验机制**：入库前检查分数范围、字段缺失、total 与子分加和一致性。

### 4.4 企业增强层

```
company_enrich.py          # CLI 入口
company_evidence_filter.py # 证据过滤
crawler/zhihu_client.py    # 知乎搜索客户端
```

**触发调查的条件**（满足任一）：
1. `company_size_small_or_unknown` — 公司规模小或未知
2. `salary_abnormal_for_small_company` — 小公司薪资异常
3. `jd_risk_score >= 3` — JD 中的风险词累计分数
4. `job_posted_over_30d_and_hr_online` — 超30天老职位 + HR在线
5. `weak_company_identity` — 公司标识弱（无 logo、未认证等）

**知乎搜索**：
- 低频小批量（`max_requests_per_run=20`）
- 只抓摘要，不抓全文
- 结果缓存 7 天（`cache_ttl_hours=168`）
- 三级风险词：强 / 中 / 弱

### 4.5 Cookie 管理层

```
cookie_manager.py            # 可复用模块（CLI 和前端都调用）
rotate_liepin_cookies.py     # 复用轮换调度
tools/refresh_liepin_cookies.py  # 半自动登录刷新
check_liepin_messages.py     # 消息接口检查
```

#### `cookie_manager.py` 核心函数

| 函数 | 用途 |
|------|------|
| `scan_and_cleanup(platform)` | 扫描 cookies/ 目录，清理过期文件，按手机号去重，同步 DB |
| `get_usable_profiles(platform)` | 查可用 cookie（ready + cooldown 已过期），按优先级排序 |
| `mark_profile_used(platform, name)` | 更新使用计数（跨天自动清零） |
| `mark_profile_blocked(platform, name)` | 标记为 needs_manual_verify |
| `mark_profile_cooldown(platform, name, hours)` | 设为冷却状态 N 小时 |
| `get_earliest_cooldown_expiry(platform)` | 查最早冷却恢复时间 |

#### `rotate_liepin_cookies.py` — 复用轮换

**核心逻辑（while 循环）**：

```
while pending_jobs > 0:
    cookie = pick_best_available_cookie()   # 按优先级选
    if not cookie:
        报告"全部冷却中，最早恢复时间 X" → 退出
    run_detail_batch(cookie, 25条)
    mark_profile_used()       # 更新计数器
    mark_profile_cooldown(2h) # 设冷却
    自动检查后处理触发条件
```

**优先级**：ready > cooldown 已过期，然后今天用得少的 > 最久没用的。

**复用**：2 小时后同 cookie 自动恢复可用，无需手动操作。

### 4.6 后处理管线

```
postprocess_pipeline.py    # 串联入口
```

依次执行 `clean_jobs.py → company_enrich.py → score_jobs.py`，支持 `--skip-*` 跳过任意阶段。

## 6. Cookie 管理体系

### 6.1 Cookie 生命周期

```
登录获取                使用中              触发风控
   │                     │                    │
   ▼                     ▼                    ▼
ready ──run──► (cooldown ──2h──► ready)    needs_manual_verify
                                         (需人工验证恢复)
                                              │
                                          disabled
```

### 6.2 文件命名规范

```
cookies/cookie_YYYYMMDD_HHMMSS_{手机号}_{platform}.json
cookies/cookie_YYYYMMDD_HHMMSS_{手机号}_{platform}_message_context.json
```

- 按手机号去重（同 profile_name 只保留最新文件）
- 超过 `cookie_max_age_days` 天自动删除

### 6.3 刷新流程

```
python tools/refresh_liepin_cookies.py --phone 138xxxx --auto
  → 打开浏览器 → 人工完成登录 → 自动检测 auth cookie → 导出
```

## 7. 配置参数一览

```python
# config.py — RUN_CONFIG

"auto_postprocess": False,           # rotate 时自动触发后处理
"auto_postprocess_min_jobs": 50,     # 触发阈值

"cookie_max_age_days": 2,            # Cookie 文件保留天数
"cookie_max_per_run": 3,             # 单次最多用几个 cookie
"cookie_cooldown_hours": 2,          # 🆕 每批后冷却时间
"cookie_daily_max_detail": 0,        # 🆕 单 cookie 每日上限（0=不限）

"list": {
    "page": 0, "pages": 1,           # 列表页范围
    "store_top_n": 30,               # 每页保留职位数
    "min_delay": 60, "max_delay": 120, # 翻页间隔
}

"detail": {
    "max_detail": 25,                # 单批最大条数
    "min_delay": 45, "max_delay": 90, # 详情间隔
    "startup_cooldown_min": 120,      # 启动冷却
    "cooldown_every": 10,            # 每 N 条额外冷却
    "cooldown_min": 180, "cooldown_max": 240,
}

# AI_CONFIG
"model": "deepseek-chat",            # DeepSeek 模型
"batch_size": 50,                    # 评分批次
"max_days_since_update": 14,         # 只评 14 天内的职位
```
